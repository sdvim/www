A CLOSER LOOK

Background updates

Something is bound to go wrong when a developer is 3000 miles away from the production machines. At any given moment there was likely foot traffic, so this would have to be stealthy.

Instead, I wrote a simple script that checked for updates every minute, and would reload the changes once the machine is idle for two minutes.

This was done by storing the codebase on Dropbox, in which myself and the four production machines all had access to. I push changes, the kiosks download it, then compare the timestamps of the current code being used against the code in the local Dropbox folder, and reload the app if no one is around.

This proved invaluable as the touchscreens began to experience hardware issues, so adjustments to the on-screen keyboard had to be made, amongst other fixes.

Smart schedule

The schedule app was made to be as direct as possible. It would select the current day's lineup upon both boot-up and after a user walks away.

Furthermore, past events would dim slightly to better stay up to date.
